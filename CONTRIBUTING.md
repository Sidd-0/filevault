# Contributing to File Vault

Thank you for your interest in contributing to File Vault!

## Development Setup

1. Fork the repository
2. Clone your fork: `git clone https://github.com/yourusername/filevault.git`
3. Create a branch: `git checkout -b feature/your-feature`
4. Make your changes
5. Run tests: `./test.sh`
6. Commit: `git commit -m "Add your feature"`
7. Push: `git push origin feature/your-feature`
8. Create Pull Request

## Code Style

### Go (Backend)
- Follow Go conventions
- Use `gofmt` for formatting
- Add comments for exported functions
- Write tests for new features

### JavaScript (Frontend)
- Use consistent indentation (2 spaces)
- Use meaningful variable names
- Add PropTypes for components
- Follow React best practices

## Commit Messages

Use clear, descriptive commit messages:
- `feat: Add user authentication`
- `fix: Resolve file upload bug`
- `docs: Update API documentation`
- `refactor: Improve deduplication logic`

## Testing

- Write tests for new features
- Ensure all existing tests pass
- Test with Docker Compose before submitting
